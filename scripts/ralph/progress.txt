## Codebase Patterns
- Budget guard: `readBudgetConfig(projectDir)` reads `budget:` section from config.yaml (per_session_usd/per_phase_usd/per_day_usd); `checkBudget(input)` returns `ok({allowed, limitType, ...})`; daily spend persisted to `.buildpact/budget-usage.json`
- Wave executor: `executeWave(tasks)` dispatches all tasks in parallel (via stub in Alpha), `executeWaves(waves)` iterates waves sequentially with halt-on-failure; both exported from src/engine/wave-executor.ts
- AuditOutcome type only allows 'success' | 'failure' | 'rollback' — no 'partial'; use 'failure' for partial failures
- buildWaveTaskGroups should filter out empty wave groups (.filter(tasks => tasks.length > 0)) to avoid phantom empty waves
- Active Squad loaded via readActiveSquad(projectDir): reads `active_squad` from config.yaml, then loads `.buildpact/squads/<name>/squad.yaml` for domain — returns undefined if 'none' or files missing
- Web Bundle mode detected via readWebBundleMode(projectDir): looks for `mode: web-bundle` in config.yaml — changes clack.select → clack.text for conversational flows
- ESM `.js` extension on ALL imports: `import { ok, err } from '../contracts/errors.js'`
- `execSync` from `node:child_process` for git commands — wrap in try/catch returning Result<string>
- Mock `node:child_process` with `vi.mock('node:child_process', () => ({ execSync: vi.fn(() => '') }))` in tests that touch git
- `Result<T, CliError>` for all fallible functions — never `throw`
- `ERROR_CODES.*` constants in `src/contracts/errors.ts` — add new codes here, never inline strings
- `ok()` / `err()` helpers — never raw `{ ok: true, value }` objects
- Named exports only — no `export default`
- Audit logging FIRST — log intent before writing files
- All user strings via `I18nResolver.t()` — never hardcoded text
- Factory function pattern for test setup — `const opts = () => ({...})`
- Test imports from implementation files — not from barrel `index.ts`
- @clack/prompts API: `spinner.stop()` takes 1 argument; use `log.success()` / `log.warn()` for display
- Layer dependency order: contracts <- foundation <- engine <- commands <- cli
- Import from sibling layer: src/engine/ imports contracts as `../contracts/` (NOT `../../contracts/`)
- noUncheckedIndexedAccess: use `record[key]!` when key is a known enum/union value; wrap const blocks with eslint-disable/enable for clean code
- Vitest 4.x for testing, mkdtemp for temp dirs, afterEach cleanup
- TypeScript strict mode, tsdown for build
- Node.js minimum 20.x, 22.x LTS recommended
- Squad scaffolding: `scaffoldSquad(name, targetDir)` creates 4-tier (T1-T4) agent templates with all 6 layers + voice DNA 5 sections + inline docs + squad.yaml + benchmark/; `validateSquadStructure` validates yaml fields + 6 layers + min 3 examples; `validateSquadSecurity` checks external URLs, executable code, path traversal, prompt injection; `installSquad` copies to .buildpact/squads/<name>/
- In runAdd tests: use `mockImplementation` not `mockResolvedValueOnce` for confirm/isCancel to avoid queue bleed across tests (clearAllMocks doesn't clear once-queues)
- Testable time: pass `nowMs = Date.now()` as last param to all time-sensitive pure functions; in tests pass a fixed epoch constant (e.g., `const NOW = 1_700_000_000_000`) for deterministic assertions
- Hyphenated `CommandId` values (e.g., `'export-web'`) must be quoted in both the type union and the REGISTRY object key: `'export-web': () => import(...)`
- Injectable fetch pattern: pass `fetchFn: typeof fetch = fetch` as last param to I/O functions that call fetch; tests inject a `vi.fn()` mock — avoids global fetch mocking
