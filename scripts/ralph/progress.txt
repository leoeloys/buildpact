## Codebase Patterns
- Active Squad loaded via readActiveSquad(projectDir): reads `active_squad` from config.yaml, then loads `.buildpact/squads/<name>/squad.yaml` for domain â€” returns undefined if 'none' or files missing
- Web Bundle mode detected via readWebBundleMode(projectDir): looks for `mode: web-bundle` in config.yaml â€” changes clack.select â†’ clack.text for conversational flows
- ESM `.js` extension on ALL imports: `import { ok, err } from '../contracts/errors.js'`
- `execSync` from `node:child_process` for git commands â€” wrap in try/catch returning Result<string>
- Mock `node:child_process` with `vi.mock('node:child_process', () => ({ execSync: vi.fn(() => '') }))` in tests that touch git
- `Result<T, CliError>` for all fallible functions â€” never `throw`
- `ERROR_CODES.*` constants in `src/contracts/errors.ts` â€” add new codes here, never inline strings
- `ok()` / `err()` helpers â€” never raw `{ ok: true, value }` objects
- Named exports only â€” no `export default`
- Audit logging FIRST â€” log intent before writing files
- All user strings via `I18nResolver.t()` â€” never hardcoded text
- Factory function pattern for test setup â€” `const opts = () => ({...})`
- Test imports from implementation files â€” not from barrel `index.ts`
- @clack/prompts API: `spinner.stop()` takes 1 argument; use `log.success()` / `log.warn()` for display
- Layer dependency order: contracts <- foundation <- engine <- commands <- cli
- Vitest 4.x for testing, mkdtemp for temp dirs, afterEach cleanup
- TypeScript strict mode, tsdown for build
- Node.js minimum 20.x, 22.x LTS recommended
